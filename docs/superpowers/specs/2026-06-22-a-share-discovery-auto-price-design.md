# A 股标的发现与自动起始价设计

## 背景

当前 Web 回测页需要用户手动输入股票代码和起始价格。这个体验不够专业：

- 用户输入 `sh600519` 后不知道这是哪只股票。
- 起始价格需要用户自己填写，但普通用户不知道该从哪里获取。
- 页面没有热门标的入口，用户只能凭记忆输入代码。

本设计将输入体验升级为“A 股热门标的发现 + 股票名称识别 + 起始价格自动化”。

## 目标

- 输入股票代码后显示股票名称，例如 `贵州茅台 · sh600519`。
- 默认不要求用户填写起始价格。
- 回测优先使用真实历史行情，并从历史行情首日收盘价生成起始价格说明。
- 增加 A 股热门股区域，默认展示成交额榜 Top 10。
- 实时接口失败时展示内置热门股 fallback，页面不能空。
- 保留高级设置，允许专业用户手动覆盖起始价格。

## 非目标

- 不做美股或港股热门榜。
- 不做完整股票搜索/模糊搜索。
- 不做行情订阅或秒级刷新。
- 不新增数据库、用户收藏或持久化配置。
- 不重构为前端框架。

## 用户流程

1. 页面加载后请求 `/api/hot-stocks`。
2. 页面展示 A 股成交额热门股；如果实时接口失败，展示备用热门池。
3. 用户点击热门股，代码输入框填入规范代码，例如 `sh600519`，旁边显示股票名称。
4. 用户也可以手动输入代码；输入变化后请求 `/api/stock/lookup` 解析名称。
5. 起始价格默认自动，由历史行情首日收盘价决定。
6. 用户展开“高级设置”后，可以手动覆盖起始价格。
7. 点击运行回测时，前端传递 `auto_price=true`；如果用户启用手动覆盖，则传递 `auto_price=false&start_price=<value>`。

## API 设计

### `GET /api/hot-stocks`

返回：

```json
{
  "source": "realtime",
  "items": [
    {
      "symbol": "sh600519",
      "code": "600519",
      "name": "贵州茅台",
      "latest": 1500.0,
      "change_pct": 1.23,
      "turnover": 1234567890.0
    }
  ]
}
```

`source` 可为：

- `realtime`：来自 akshare `stock_zh_a_spot_em()`。
- `fallback`：实时接口失败或返回空数据时使用内置热门池。

### `GET /api/stock/lookup?symbol=sh600519`

返回：

```json
{
  "symbol": "sh600519",
  "code": "600519",
  "name": "贵州茅台",
  "latest": 1500.0,
  "source": "realtime"
}
```

未识别时仍返回规范代码：

```json
{
  "symbol": "sh600519",
  "code": "600519",
  "name": null,
  "latest": null,
  "source": "unknown"
}
```

### `GET /api/backtest`

新增参数：

- `auto_price: bool = true`
- `use_real_data: bool = true`
- `start_price: Optional[float] = None`

行为：

- `use_real_data=true` 时优先调用 `portfolio.data.fetch_hist(symbol, days)`。
- 如果真实历史行情可用，回测使用真实 OHLCV 数据。
- 起始价格说明使用历史行情首日收盘价，返回 `price_source: "historical_first_close"`。
- 如果真实行情失败，回退到 `generate_stock_data()`，返回 `data_source: "simulated_fallback"`。
- 如果 `auto_price=false` 且 `start_price` 有值，模拟 fallback 使用该价格，返回 `price_source: "manual_override"`。

## 前端设计

### 参数区

- 将“股票代码”升级为“标的”输入。
- 输入框下方展示识别结果：
  - 已识别：`贵州茅台 · sh600519`
  - 未识别：`未匹配名称 · sh600519`
- “起始价格”从主参数条移除。
- 新增只读价格来源提示：`起始价：自动 · 历史首日收盘价`。
- 新增“高级设置”折叠区：
  - 开关：手动覆盖起始价格
  - 输入：起始价格

### 热门股区域

放在参数条下方、KPI 上方：

- 标题：`热门 A 股`
- 右侧来源标识：`实时成交额榜` 或 `备用热门池`
- 每个标的显示：
  - 股票名称
  - 规范代码
  - 最新价
  - 涨跌幅
  - 成交额
- 点击热门股后填入代码并刷新股票名称显示。

### 洞察栏

增加标的信息：

- 股票名称和代码
- 数据来源：真实历史行情 / 模拟 fallback
- 起始价格来源：历史首日收盘价 / 手动覆盖 / 模拟默认值

## 测试策略

- 纯函数测试股票代码规范化。
- 测试实时热门股 fetcher 抛错时返回 fallback。
- 测试股票 lookup 能从 fallback 识别名称。
- 测试 `auto_price=true` 时回测 API 不再要求 `start_price`。
- 测试页面 HTML 包含热门股容器、高级设置和股票名称展示区域。

## 验收标准

- 用户不填写起始价格也能运行回测。
- 输入或点击 `sh600519` 后页面显示“贵州茅台”。
- 热门股区域正常展示；实时接口失败时展示 fallback。
- 回测结果返回 `data_source` 和 `price_source`。
- 现有测试和新增 Web 测试全部通过。
