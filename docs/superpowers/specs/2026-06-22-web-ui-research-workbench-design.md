# Web UI 研究工作台重设计

## 背景

当前 Web 页面完成了基础回测能力：策略参数输入、指标卡片、Plotly 图表渲染和策略说明。问题是视觉和信息架构偏 demo：控件、说明、指标和图表依次堆叠，缺少专业分析工具的层次、信息密度和结果解读区域。

本设计将页面改为“研究工作台”风格，并采用用户确认的“右侧洞察栏”布局。

## 目标

- 让单页 Web UI 看起来像策略研究工具，而不是演示页。
- 保留当前功能和 API，不扩展后端能力。
- 提升首屏信息组织：参数、关键指标、主图、策略洞察同时可见。
- 让策略说明、风险摘要、参数摘要拥有固定位置，辅助用户理解回测结果。
- 保持实现范围可控，继续使用 `web/app.py` 中的内嵌 HTML/CSS/JS。

## 非目标

- 不新增真实行情数据接入。
- 不新增登录、项目管理、多页面导航或持久化保存。
- 不新增策略对比页面。
- 不拆分前端构建系统，不引入 React/Vue 等框架。

## 视觉方向

采用白底、低饱和、研究工作台风格：

- 背景：浅灰工作区，弱化装饰。
- 面板：白色卡片、细边框、轻量圆角。
- 主色：专业蓝用于主操作和图表重点。
- 状态色：绿色表示正收益/有效信号，红色表示亏损/风险，琥珀色表示中性风险指标。
- 字体层级：紧凑、清晰，避免 hero 式大标题。

整体气质偏“投研报告 + 交互分析工具”，而不是营销页面或深色交易终端。

## 页面结构

页面采用单页布局：

1. 顶部应用栏
   - 产品名：`Quant Research Workbench`
   - 副标题：策略回测、模拟数据、Plotly 交互图表
   - 右侧状态：数据就绪、版本信息

2. 运行参数条
   - 策略选择
   - 股票代码
   - 数据天数
   - 起始价格
   - 初始资金
   - 运行回测按钮

3. 核心指标带
   - 年化收益率
   - 总收益率
   - 最大回撤
   - 胜率
   - 交易次数
   - 最终资产

4. 主分析区
   - 左侧：Plotly 主图，包含价格走势、交易信号、成交量和资产曲线。
   - 右侧：洞察栏，包含策略逻辑、风险摘要、参数摘要和数据说明。

## 交互行为

- 页面加载后继续自动调用 `/api/backtest/demo` 渲染默认结果。
- 点击“运行回测”后继续调用 `/api/backtest`。
- 图表继续使用 `renderChart()` 和 `Plotly.newPlot()`，不回退到 `innerHTML` 注入 Plotly script。
- 指标卡继续由 `renderMetrics()` 渲染，但展示顺序改为更适合首屏阅读的核心 6 项。
- 右侧洞察栏在当前阶段使用前端已有策略描述和当前参数生成展示内容，不依赖后端新增字段。
- 错误状态仍显示在图表区域，但样式与新面板一致。

## 响应式设计

- 桌面端：参数条、指标带、主图和右侧洞察栏同屏展示。
- 中等宽度：参数条和指标带自动换行，主图仍优先占据主要宽度。
- 移动端：右侧洞察栏下移，参数控件两列或单列排列，确保文字不溢出。

## 实现范围

主要修改 `web/app.py` 的 `_PAGE_HTML`：

- 重写 CSS 变量、布局类和组件样式。
- 重排 HTML 结构。
- 调整 `renderMetrics()` 生成的指标卡样式和展示数量。
- 增加 `renderInsights()` 或同等前端函数，更新右侧洞察栏内容。
- 保留当前 API、后端回测逻辑和 Plotly JSON 渲染路径。

测试范围：

- 保留现有 Web payload 测试。
- 增加或更新测试，确保页面包含新的关键容器和 `Plotly.newPlot()` 渲染路径。
- 运行完整测试套件确认回测逻辑无回归。

## 验收标准

- 首屏不再呈现 demo 式堆叠布局。
- 用户无需滚动即可看到运行参数、核心指标、主图和策略洞察。
- 图表仍能正常渲染，不出现空白容器。
- 页面在桌面和窄屏下没有明显文字溢出或布局重叠。
- `tests/run_all.py` 通过。
